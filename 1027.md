### 아이디어

출발 점정 V0로부터 가장 가까운 순서로 다른 정점들까지의 최단 경로를 찾는다.

1. n 번째 반복이 시간되기 전에 v0에 가장 가까운 (n-1)개의 정점들까지의 최단 경로를 찾는다. 
2. V0이 정점들과 최단 경로들 상의 간선들은 주어진 그래프의 부분 트리 Ti를 형성한다.

### 진행과정

![Untitled](https://user-images.githubusercontent.com/55980680/97357116-9ebd2600-18dc-11eb-83be-cce011a455e7.png)


1단계 : 

노드 0의 가장자리 정점은 1,4이고 

       `dist[1] = min(dist[1],dist[0]+len(0,1))` 

       `dist[4] = min(dist[4],dist[0]+len(0,4))` 

0에서 1까지의 거리는 1의 값( 1의 값은 트리 T에 들어갔을 경우에 생성되기 때문에 지금은 값이 0이다.) 과 0의 값과 0과 1사이의 값을 비교하여 갱신해준다. 

 dist[1] = 2, dist[4] = 7 이기 때문에 pix[0]=1,  dist[1] = 2 를 트리로 넣어준다.

2단계 :

 트리T의 가장자리 정점은 2,4 이다. 노드 1에서 2,4,까지의 거리를 구한다.

         `dist[2] = min(dist[2],dist[1]+len(1,2))`

         `dist[4] = min(dist[4],dist[1]+len(1,4))` 

dist[2]의 값은 새로 생성되고 dist[4]의 값은 1단계에서 구한 (0,4) 의 길이와 (0,1,4) 의 길이를 비교하여 더 작은 값을 담아준다.

dist[2] = 7, dist[4] = 3이기 때문에 pix[1]=4, dist[4]=3을 트리에 넣어준다.

3단계 :

트리T의 가장자리 정점은 3,2이다. 노드 4에서 3,2까지의 거리를 구한다.

     `dist[2] = min(dist[2],dist[4]+len(4,2))`
  
     `dist[3] = min(dist[3],dist[4]+len(4,3))` 

dist[3]의 값은 새로 생성되고 dist[2] 의 값은 2단계에서 구한 값 7(0,1,2)와 (0,1,4,3)의 길이를 비교하여 더 작은 값을 담아준다.

dist[2] = 6, dist[3] = 9 이기 때문에 pix[2] = 2, dist[2] = 6를 트리에 넣어준다.

4단계:

 트리 T의 가장자리 정점은 3이다. 노드 2에서 3까지의 거리를 구한다.

    `dist[3] = min(dist[3],dist[2]+len(2,3))` 

dist[3]의 값은 2단계에서 구한값 ( 0,1,4,3) 값과 (0,1,4,2,3) 값을 비교해서 더 길이가 작은 값을 담아준다. 

dist[3]은 9 이기 때문에 pix[3]=3, dist[3]= 9 ,  (4,3) 의 간선을 트리에 담아준다.

### 유사코드

```java
//T : 트리 , n: 정점의 갯수 
//  w[a,b] : a와 b의 간선길이
// d[] : 갱신되는 값, p[] 트리에 고정으로 들어가는 값

1. T 초기화

for(i=0;  i<n; i++){
	d[i] = INF;
	p[i] = -1;
}
d[0] = 0;

while( T < n){
			T에 포함되지 않은 정점 중 d[i]가 가장 작은 가장자리 정점 u 선택
			T = T ∪ (u)
			for(int v=0; v<u; v++){
				//새로 추가된 정점u의 가장자리 정점들의 값 계산
					if(v∉ T && w[u,v] ≠ 0 && d[u] < ∞ && d[u] + w[u,v] <d[v]){
							d[v] = d[u] + w[u,v]
							p[v] = u
					}
	}
```

기본연산 : 

`if(v∉ T && w[u,v] ≠ 0 && d[u] < ∞ && d[u] + w[u,v] <d[v])`-> n(-1)번 수행

`while( T < n`-> n번 수행

시간복잡도 n^2
