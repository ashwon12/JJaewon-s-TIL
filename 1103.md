## Algorithm - 문제의 분류

세상의 문제를 풀 수 있는 문제와 풀 수 없는 문제로 분류하게 된다.

- **풀수 있는 문제**
    1. NP : 다항 시간 내에 풀 수 없는 문제, 다른 말로는 비결정론적 가항 시간 알고리즘으로 해결할 수 있는 문제라고도 부른다.
    2. P : 다항 시간 내에 풀 수 있는 문제 , 시간복잡도가 다항식인 문제를 말한다.
    3. NP-완전 문제 : 지수 시간 내에 풀 수 있는 문제

- **풀 수 없는 문제**

    예 : 정지문제 

    프로그램 P와 입력 x가 있는데 프로그램에 x를 입력했을 경우 궁극적으로 종료하는가? 라는 질문을 던졌을 때 yes,no로 대답 할 수 있는 프로그램이 존재하지 않는다.

또 다른 분류방법으로는 결정문제와 최적화 문제로 분류하는 방법이 있다.

- **결정문제**

    답이 yes, 또는 no인 문제가 결정 문제

    예 ) 무방향 그래프의 정점 s에서 f까지의 가는 경로의 거리가 k(상수) 이하인 경로가 있는가?!

    답 ) yes 또는 no

- **최적화 문제**

    최적의 해를 요구하는 문제

    최적화 문제는 모두 대응하는 결정 문제로 바꾼 다음에 해결해야 한다. 그렇기 때문에 결정문제보다는 조금 어렵다.

    예) 무방향 그래프에서 정점 s에서 f까지의 가는 최단 경로는 얼마인가?

    답 ) 최적화 답

## NP 문제 - 다항 시간 내에 풀 수 없는 문제

비결정론적 다항 시간 알고리즘에 의해 해결할 수 있는 결정 문제들의 집합을 NP라고 한다.

🤔**비결정론적 다항 시간 알고리즘은 무엇인가?**

한 **결정 문제**의 사례 I를 입력을 받아서 

1. 주어진 사례 I에 대한 해 S를 추측한다. ⇒ **비결정론적 '추측'단계**
2. 사례 I와 S를 입력받아 옳은 해라면 Yes, 아니면 No or 알고리즘 이 종료되지 않게 한다. ⇒ **결정론적 '확인' 단계**

위와 같은 2단계의 알고리즘을 수행하는 것.

예를 들어서 설명해보자면 , 결정문제인 정렬문제를 살펴보자.

배열 A = {5,3,2,9,1,2,5} 이 존재할 때 이 배열을 정렬하는 사례 I를 든다.

사례 I = { 1,2,2,3,5,9,5} 라고 할 때 이 배열은 정렬되어 있지 않기 때문에 해가 'NO'가 된다.  사례 I가 정렬이 되어 있는지 확인하는 과정에서 **시간복잡도 O(n)**이 발생하기 때문에 ***다항 시간 알고리즘***이다. 따라서 이 문제는 NP 문제이다.

## NP완전 문제 - 지수 시간 내에 풀 수 있는 문제

NP에 속하는 문제이고 어떤 NP완전문제를 다항 시간 알고리즘에의해 풀 수 있다면 나머지 모든 NP완전문제들도 다항 시간 알고리즘에 의해 풀 수 있다!

NP완전문제는 결정문제에 국한되어 있다는 특징이 있다.

**NP-완전의 정의**
1. 문제 D는 NP에 속한다.
2. NP에 속한 모든 문제는 다항 시간에 문제 D로 변환 가능하다.

**⁉ NP 완전 문제의 핵심 원리 ⁉**

결정 문제의 변환을 이용한 해결 과정

💡 결정 문제 A를 다항 시간에 해결할 수 있는지 알고 싶다!

**가정**

   - 다항 시간에 해결가능한 결정 문제 B를 이미 알고 있다.
   - 문제 A를 다항 시간에 문제 B로 변환가능, 문제 B의 해는 문제A의 해와 항상 일치

**문제 해결**

1. 문제 A의 사례를 문제 B로 변환한다.
2. 변환된 문제B의 사례를 문제 B의 알고리즘을 이용하여 해결한다.

👉 **해밀토니안 회로문제 → 외판원 문제로 변환**

해밀토니안 회로 문제 : 무방향 그래프에 해밀토니안 회로(무방향 그래프의 모든 정점을 정확히 한 번씩만 방문하고 출발 정점으로 돌아오는 경로)가 있는가?

외판원 문제 : 양의 가중치를 갖는 무방향 **완전 그래프**에서 거리가 K 이하인 해밀토니안 회로가 있는가?

완전 그래프에서는 해밀토니안 회로가 반드시 존재하게 된다.

**⇒ 해밀토니안 회로 문제의 사례를 외판원 문제의 사례로 다항 시간에 변환**
