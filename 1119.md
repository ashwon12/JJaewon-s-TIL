## IPC( 프로세스간 통신) : sochet programming

### Sochets Programming?

다른 프로세스와 메세지를 보내고 받을 수 있는 ***특정 메모리 영역***이다.
 만약에 플로그를 소켓에 연결하면 어떻게 발전소로부터 소켓까지 전달되는지 알 수 없지만 전기를 공급받게 된다. 
이것처럼 다른 프로세스의 **소켓 메모리 영역과 통신을 하게 되면 어떠한 규격으로 연결하는지 정확하게 알 지 못해도 데이터를 전달 받을 수 있다.**

<br>

**정의**
```txt
네트워크를 이용하여 연결된 다른 프로세스와 메세지를 송수신할 수 있도록 운영체제에서 도와주는 인터페이스 
```

<br>

**기술적 의미**

     - 소켓번호 = ip 주소+포트 

    ⇒ 네트워크에 연결되어 있는 전 세계 유일의 특정 포트
    
    
   
<br><br>
### Socket

소켓은 유닉스시스템에 속한 프로세스 뿐만 아니라 서로 다른 유닉스시스템들에 속한 프로세스들 간의 통신도 가능하게 된다. 
pipe는 단일 컴퓨테어만 사용이 가능한 반면 소켓은 매우 범용성이 크다는 장점이 있다.

특징

  - 프로세스간의 통신을 위한 통신의 끝점(또는 출입구)역할을 한다.

  - 디스크립터로 접근 가능

  - 전혀 관계없는 프로세스간 통신을 위해 이름을 붙인다. ⇒ 파일 이름 또는 파일 디스크립터 번호 사용

  - bind() 함수 사용 

       ⇒ 소켓에 특정 번호를 엮어 놓느 작업으로 어느 프로세스든지 번호를 알면 통신이 가능하게 하도록 하기 위함

<br><br>

### stream socket을 사용한 IPC

🤔**stream socket이란??**

  Virtual circuit(가상회로) 기능을 이용하여 구현한 소켓이다. 
  프로세스간 **대량의 데이터를 송수신** 하는 경우 이용하고,
  통신의 **신뢰성 보증**, 송신한 정보가 없어지거나 바뀌지 않고,
  **올바른 순서로 수신**된다.
  
  <br>

- **클라이언트**

    Socket 작성 : **socket()** 
    
    Socket에 이름 붙임 : **bind()** →  생략 가능
    
    Server에 접속 요구 : **connect()**
    
    데이터 송수신 : **read()/write()** 
    
    ⇒socket이 파일 디스크립터로 접근이 가능하기 때문
                             
    Socket 제거 : **close()**

- **서버**

    Socket 작성 : **socket()** 
    
    Socket에 이름 붙임 : **bind()** 
    
    클라이언트 접속 요구 받을 준비 : **listen()**
    
    접속 요구 허가 : **accept()**
    
    데이터 송수신 : **read()/write()**
    
    Socket 제거 : **close()**
    
 <br>

### Datagram socket을 사용한 IPC

🤔**Datagram socket이란??**

  Datagram 기능을 이용하여 구현한 소켓이다.
  프로세스간에 ***작은 데이터를 계속해서 보내는 통신***으로,
  매번 데이터를 보낼 때마다 상대방을 지정한다.
  **신뢰성이 보증되지 않는 단점**이 있다.
  
  <br>

- **클라이언트**
    Socket 작성 : socket() 
    
    Socket에 이름 붙임 : bind() →  생략 가능
    
    데이터 송신 : sendto()
    
    데이터 수신 : recvfrom()
    
    Socket 제거 : close()

- **서버**

    Socket 작성 : socket() 
    
    Socket에 이름 붙임 : bind()
    
    데이터 수신 : recvfrom()
    
    데이터 송신 : sendto()
    
    Socket 제거 : close()
