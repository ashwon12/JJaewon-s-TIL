## mongoDB

NOSQL문의 데이터베이스로 딕셔너리 형태로 데이터를 저장해 준다.  

`DATABASE > Collection > Document > Field`  의 크기로 데이터가 저장되어 있다.

스키마가 존재하지 않고 collection 내의 Document가 각각 다른 스키마를 가질 수 있다.

### 파이썬에서 mongoDB 사용하기

```jsx
from pymongo import MongoClient  

client = MongoClient('localhost', 27017) 
db = client.databaseName
```

- pymongo 패키지를 먼저 설치해주어야 한다. ( pip install pymongo )
- mongoDB에서는 추가적인 설정을 하지 않으면 포트번호 27017에서 돌아간다.
- 기존에 있는 이름을 만들면 그 곳에 연결이 되고 그게 아니면 새로 생성된다.

### create()

```python
# 'users'라는 collection에 데이터를 생성합니다.
# 한 줄이 document이다.
db.users.insert_one({'name': '덤블도어', 'age': 116})
db.users.insert_one({'name': '맥고나걸', 'age': 85})
db.users.insert_one({'name': '스네이프', 'age': 60})

user1 = {'name': '론', 'age': 40}
user2 = {'name': '해리', 'age': 40}
db.users.insert_one(user1)
db.users.insert_one(user2)
```

### read()

- find()

```python
# MongoDB에서 데이터 모두 보기
all_users = list(db.users.find({}))

print(all_users[0])  # 0번째 결과값을 보기
print(all_users[0]['name'])  # 0번째 결과값의 'name'을 보기

for user in all_users:  # 반복문을 돌며 모든 결과값을 보기
    print(user)
```

find는 list로 받아오기 때문에 배열 안에 딕셔너리가 들어간다. 그렇기 때문에 2차원배열처럼 값을 입력해주어야 확인가능!

- find_one()

```python
user = db.users.find_one({'name': '덤블도어'}, {'_id': False})
print(user)

/*첫번째 딕셔너리는 필드값이 일치하는 얘들 가져오기
두번째 딕셔너리를 필드값이 일피하는얘들만 빼고 가져오기*/
```

하나만 가져오기 때문에 딕셔너리형식으로 가져오게 된다.

### update

[더 많은 값들 확인](https://docs.mongodb.com/manual/reference/operator/update/) 

```python
#하나만 업데이트
db.users.update_one({'name': '덤블도어'}, {'$set': {'age': 19}})

#여러개 업데이트
db.users.update_many({'name': '덤블도어'}, {'$set': {'age': 19}})
```

여러개를 업데이트 할 경우 update_many를 사용하는 것이 훠~~ㄹ씬 효율적

# git

1. `git init` , git이 해당 디렉토리에 파일을 관리할 수 있게 해줌
2. `git remote add origin <저장소 url>` 

    해당 파일과 저장소를 연결하고 그 저장소를 origin이라고 부를거다

3. `git remote get -url origin`

    해당 파일과 연결된 저장소 url 알아내는 명령어

### git file lifecycle


![Untitled](https://user-images.githubusercontent.com/55980680/96758801-32d94a00-1412-11eb-80b4-632a3f549aae.png)


1. 파일 생성 → untracked 상태
2. `git add <filename>` → Staged 상태, 커밋 가능한 상태
3. `git commit -m "commit message"` → Unmodified 상태
4. `git push origin master` → master에 있던 변경사항들 저장소에 올라감
5. 파일 수정 → modified 상태, 커밋 불가능, add 해줘야함
